# kvy[ɪk] — описание проекта

**Корпоративный поиск знаний, по запросу.**

kvy[ɪk] (читается как «quick», `[kwɪk]`) — сервис автоматизированного обучения
RAG-моделей: корпоративная платформа для интеллектуального поиска знаний,
которая отвечает на вопросы сотрудников по внутренним документам компании —
со ссылками на источники и за секунды.

---

## Проблематика

Осознание проблемы пришло — а готового решения под задачу **на рынке нет**.
Именно этот пробел закрывает kvy[ɪk].

1. **Знания заперты в архивах.** Документы и экспертиза — главный актив компании.
   Но он лежит мёртвым грузом в разрозненных порталах, базах и файловых хранилищах.
2. **Время утекает на поиск.** Сотрудники тратят часы на поиск нужного вместо работы.
   Отставание в скорости доступа к информации = отставание в скорости бизнеса.
3. **Экспертиза уходит с людьми.** Увольняется специалист — компания теряет его знания.
   Новые сотрудники остаются без доступа к опыту лучших.
4. **Технологии есть — продукта нет.** LLM и RAG уже созрели, но экспертов
   катастрофически не хватает, а готового решения под корпоративный контур не найти.

> Связка **LLM + RAG** сокращает время до нужного ответа с десятков минут до секунд:
> **10 мин → 8 сек**.

---

## Цель проекта

Создание корпоративной платформы для интеллектуального поиска знаний
и автоматизированного обучения RAG-моделей.

## Основные задачи

- Проанализировать существующие решения в области RAG.
- Разработать архитектуру платформы.
- Настроить векторное хранилище данных.
- Интегрировать языковые модели.
- Провести тестирование решения.

---

## Архитектура

Поток данных от пользователя до ответа:

```
OpenWebUI        — интерфейс пользователя
   ↓
RabbitMQ         — очереди задач и индексации
   ↓
ChromaDB         — векторное хранилище embeddings
   ↓
Ollama + Gemma   — локальная LLM, генерация ответа
   ↓
Пользователь     — ответ со ссылками на источники
```

**Технологический стек:** OpenWebUI, Ollama, Gemma, ChromaDB, RabbitMQ, Docker.
